[ PROBLEM A : Order Book Depth (Range Sum Queries) ]

  Scenario : N=500, Q=1000, 80% point queries (L==R)
  Note     : Brute: 1 array access per query. Fenwick: 2*prefix_sum = ~18 bit-ops. Sqrt: scans 1 element in same-block branch. Brute wins on minimal-range queries.
  Brute   O(N)     :        2 us  PASS
  Sqrt    O(sqrtN) :        4 us  PASS
  Fenwick O(logN)  :       15 us  PASS
  FASTEST  : Brute

  Scenario : N=50000, Q=2000, tiny range queries (width 1-3 elements)
  Note     : Brute scans 1-3 elements. Fenwick always pays 2*log2(50000)=32 ops. Sqrt same-block scan: 1-3 elements. Brute or Sqrt beat Fenwick constant overhead.
  Brute   O(N)     :       12 us  PASS
  Sqrt    O(sqrtN) :       14 us  PASS
  Fenwick O(logN)  :       36 us  PASS
  FASTEST  : Brute

  Scenario : N=100000, Q=1000, adversarial full-range queries [1,N]
  Note     : Brute scans 100K elements per query. Sqrt traverses all 317 blocks + partial edges. Fenwick: 17 pointer-hops. Only Fenwick survives.
  Brute   O(N)     :    15237 us  FAIL
  Sqrt    O(sqrtN) :      180 us  PASS
  Fenwick O(logN)  :       11 us  PASS
  FASTEST  : Fenwick

  Scenario : N=100000, Q=1000, 90% updates / 10% queries
  Note     : Sqrt update O(1): write vol[idx] + block_sum[idx/B]. Fenwick update: cascade through 17 BIT nodes. Update-heavy load: Sqrt wins.
  Brute   O(N)     :     1122 us  PASS
  Sqrt    O(sqrtN) :       22 us  PASS
  Fenwick O(logN)  :       16 us  PASS
  FASTEST  : Fenwick

  ENGINE VERDICT (live market: N=100k, Q=1000, mixed ops)
  Selects : Fenwick Tree (O(log N) per operation)
  Reason  : Fenwick tree (O(log N)) operations ~ 16609; guarantees safe execution even with large N and Q.

[ PROBLEM B : Triangular Arbitrage (Negative Cycle Detection) ]

  Scenario          : V=1000, E=999, linear chain (w=-1, no neg-cycle)
  Note              : SPFA: chain processed in queue order; each node dequeued once -> O(V+E)=2000 ops. BF: 1 hop relaxed per pass; needs V-1=999 passes x 999 edges = 1M ops. SPFA wins ~500x.
  BF Brute  O(VE)   :        4 us  PASS  neg-cycle=NO
  SPFA              :       93 us  PASS  neg-cycle=NO
  BF Opt(early-exit):        6 us  PASS  neg-cycle=NO
  FASTEST           : BF Brute

  Scenario          : V=400, E=50000, dense ALL-POSITIVE weights
  Note              : dist[]=0; all w>0: no edge ever relaxes. BF Opt: 1 pass x E edges, updated=false, exits immediately. SPFA: dequeues all 400 nodes, checks ~50K adj entries with queue overhead. BF Opt wins.
  BF Brute  O(VE)   :       82 us  PASS  neg-cycle=NO
  SPFA              :      869 us  PASS  neg-cycle=NO
  BF Opt(early-exit):       45 us  PASS  neg-cycle=NO
  FASTEST           : BF Opt

  Scenario          : V=300, E=~10K, dense, obvious neg-cycle 0->1->2->0 (sum=-180)
  Note              : BF Opt: neg-cycle keeps updated=true; runs full 299 passes x ~10K edges = ~3M ops, then 1 detection pass. SPFA: cnt[0,1,2] each exceed V=300 after ~300 cycle re-queues (~900 ops). SPFA detects ~3300x faster.
  BF Brute  O(VE)   :     2280 us  PASS  neg-cycle=YES
  SPFA              :    10390 us  PASS  neg-cycle=YES
  BF Opt(early-exit):     2225 us  PASS  neg-cycle=YES
  FASTEST           : BF Opt

  Scenario          : V=300, E=40003, dense, neg-cycle buried at nodes 297-299
  Note              : Dense random graph forces SPFA to propagate through all ~300 nodes before cycle detected. BF Opt: full 299 passes x 40K edges = 12M ops (no early exit with neg-cycle). Both hit O(VE); shows SPFA's worst-case variance on dense adversarial graphs.
  BF Brute  O(VE)   :    47363 us  PASS  neg-cycle=YES
  SPFA              :    43786 us  PASS  neg-cycle=YES
  BF Opt(early-exit):    49147 us  PASS  neg-cycle=YES
  FASTEST           : SPFA

  ENGINE VERDICT (live market: V=5 currencies, E=18 pairs)

  Brute O(2^T)     : SKIPPED (T=5000 exponential, would never finish)
  Greedy O(T)      : cost=     50000         2 us  PASS
  Optimal DP O(TI) : cost=       100      1089 us  PASS

  WINNER    : Optimal DP
